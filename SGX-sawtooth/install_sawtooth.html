安装Hyperledger Sawtooth

1.安装环境

硬件：根据选用的共识算法决定。Dev mode或Poet Simulator可以使用任何PC或虚拟机。Poet on SGX需要在基于Skylake的笔记本或NUC上运行。

操作系统：Ubuntu 16.04 Server

网络：假设所有机器都在同一个网段，假定配置为静态IP。

2.软件包的安装

2.1 安装SGX驱动和SDK

如果要选用Poet on SGX，还需要安装SGX驱动。其他共识机制可以跳过这个步骤。

从ICLS client下载iclsClient-1.45.449.12-1.x86_64.rpm，复制到目标机器上。

其他需要的文件可以从01.org和JHI repo下载。

$ sudo apt-get install git libssl-dev libcurl4-openssl-dev libprotobuf-dev build-essential python uuid-dev libxml2-dev cmake pkg-config alien
$ sudo alien --scripts iclsClient-1.45.449.12-1.x86_64.rpm
$ sudo dpkg -i iclsclient_1.45.449.12-2_amd64.deb
$ git clone https://github.com/intel/dynamic-application-loader-host-interface
$ cd dynamic-application-loader-host-interface
$ cmake .; make; sudo make install; sudo systemctl enable jhi
$ cd ..
$ wget https://download.01.org/intel-sgx/linux-2.1.2/ubuntu64-server/sgx_linux_x64_driver_1bf506e.bin

每个节点都需要安装Sawtooth最新稳定版，目前为1.0.2。最简单的方法是用apt-get命令来安装，安装命令为：

$ sudo add-apt-repository 'deb http://repo.sawtooth.me/ubuntu/1.0/stable xenial universe'
$ sudo apt-get update
$ sudo apt-get install -y sawtooth python3-sawtooth-poet-sgx

如果在公司内网，需要先设置http_proxy和https_proxy环境变量

3.节点初始化

每个节点上的validator都需要一个密钥，在运行validator之前，每个节点上都要运行以下命令：

$ sudo sawadm keygen

产生的密钥在/etc/sawtooth/keys/下。

另外，检查当前节点上没有残留旧的区块链数据：

$ sudo ls /var/lib/sawtooth/

其中应该没有任何内容。

4.产生创世块

每个区块链系统都必须有一个创世块。产生创世块的步骤只需在第一个启动的validator上执行即可，validator启动后，创世块会自动复制到其他连接上来的validator中去。

根据使用的共识机制不同，使用以下三种步骤之一来产生创世块：

4.1.使用dev mode

Dev mode是Sawtooth里最简单的共识实现，用于应用程序开发过程。

$ sawtooth keygen
$ sawset genesis
$ sudo -u sawtooth sawadm genesis config-genesis.batch

然后转到第5章继续启动validator。

4.2.使用Poet Simulator

Poet simulator可以在不支持SGX的机器上模拟完整的Poet执行机制。为了创建Poet simulator的创世块，可以使用以下命令：

$ sudo -u sawtooth -s
$ cd /tmp
$ sawset genesis -k /etc/sawtooth/keys/validator.priv -o config-genesis.batch
$ sawset proposal create -k /etc/sawtooth/keys/validator.priv sawtooth.consensus.algorithm=poet sawtooth.poet.report_public_key_pem="$(cat /etc/sawtooth/simulator_rk_pub.pem)" sawtooth.poet.valid_enclave_measurements=$(poet enclave measurement) sawtooth.poet.valid_enclave_basenames=$(poet enclave basename) -o config.batch
$ poet registration create -k /etc/sawtooth/keys/validator.priv -o poet.batch
$ sawadm genesis config-genesis.batch config.batch poet.batch
$ exit

然后转到第5章继续启动validator。

4.3.使用Poet on SGX

4.3.1 申请SPID

要使用Poet on SGX，必须先创建一个自签名证书，然后用该证书申请IAS权限。先创建一个名为client.cnf的文件，包含以下内容：

[ ssl_client ]
keyUsage = digitalSignature, keyEncipherment, keyCertSign
subjectKeyIdentifier=hash
authorityKeyIdentifier=keyid,issuer
extendedKeyUsage = clientAuth, serverAuth

然后创建证书（标注***的内容按实际填写，其他地方直接按回车）：

4.3.2 配置文件

创建一个POET配置文件：

$ sudo vi poet_enclave_sgx.toml
$ sudo chmod 640 poet_enclave_sgx.toml
$ sudo chown root:sawtooth poet_enclave_sgx.toml

内容为：

# Service Provider ID. It is linked to the key pair used to authenticate with
# the attestation service.
 
spid = '上一步收到的SPID'
 
# ias_url is the URL of the Intel Attestation Service (IAS) server.
 
ias_url = 'https://test-as.sgx.trustedservices.intel.com:443'
 
# spid_cert_file is the full path to the PEM-encoded certificate file that was
# submitted to Intel in order to obtain a SPID
 
spid_cert_file = '/etc/sawtooth/sgx-certificate.pem'

另外将上一步生成的client.pem文件复制过去：

$ sudo cp client.pem /etc/sawtooth/sgx-certificate.pem
$ sudo chmod 640 sgx-certificate.pem
$ sudo chown root:sawtooth sgx-certificate.pem

5.启动第一个validator和transaction processor

打开一个新的控制台，用以下命令启动validator：

如果在公司内网且选择Poet on SGX方式运行，下面这条命令需要为sudo以后的环境设置代理。

$ sudo -u sawtooth sawtooth-validator -vv

另外，打开几个新的控制台执行以下命令。所有这些进程都需要保持运行状态才能让sawtooth正常运行：

REST API：

$ sudo -u sawtooth sawtooth-rest-api -v

Transaction Processor：

$ sudo -u sawtooth settings-tp -v
$ sudo -u sawtooth intkey-tp-python -v

如果在公司内网且选择Poet on SGX方式运行，下面这条命令需要为sudo以后的环境设置代理。

$ sudo -u sawtooth poet-validator-registry-tp -v

启动完毕后，在validator的控制台上可以看到如下日志输出：

[... INFO     processor_handlers] registered transaction processor: connection_id=..., family=sawtooth_settings, version=1.0, namespaces=['000000']
[... INFO     processor_handlers] registered transaction processor: connection_id=..., family=intkey, version=1.0, namespaces=['1cf126']
[... INFO     processor_handlers] registered transaction processor: connection_id=..., family=sawtooth_validator_registry, version=1.0, namespaces=['6a4372']

表明transaction processor都已成功连到validator上。

5.1.测试validator

运行以下命令创建一个intkey交易，用于测试：（此处会自动用到之前用sawtooth keygen创建的私钥）

$ intkey set --wait 30 testkey 123

这将在区块链上创建一个交易，把testkey的值设为123，参数--wait 30表示在发送交易后等待他的状态变为committed，最多等待30秒。然后我们可以查看testkey的值：

$ intkey list
testkey: 123

5.2.测试REST API

可以用curl命令测试REST API，默认的地址为127.0.0.1:8008。

$ curl http://127.0.0.1:8008/peers
{
  "data": [],
  "link": "http://127.0.0.1:8008/peers"
}

REST API的定义参见这里。

6.将sawtooth设置成服务

Sawtooth软件包安装时会自动创建相应的服务，但默认状态下都设定为禁用。由于每次打开多个控制台分别手工运行sawtooth的各个组件太过麻烦，我们可以让这些服务在开机时自动启动。

输入以下命令将服务打开：

$ sudo systemctl enable sawtooth-intkey-tp-python.service sawtooth-poet-validator-registry-tp.service sawtooth-rest-api.service sawtooth-settings-tp.service sawtooth-validator.service

如果在公司内网且选择Poet on SGX方式运行，还需要设置代理。编辑以下两个文件：

$ sudo vi /etc/default/sawtooth-validator
$ sudo vi /etc/default/sawtooth-poet-validator-registry-tp

在两个文件中都添加下面这两行：

http_proxy=http://ip:port
https_proxy=http://ip:oprt

然后重启机器，会发现所有的组件都自动运行了。服务输出的日志可以用journalctl命令查看。

7.创建包含多个节点的网络

到目前为止，我们配置的sawtooth区块链还只有一个节点。这种配置只适用于开发应用时的本地测试，在实际应用环境中，我们通常需要部署多个节点，每个节点都运行相同的validator和transaction processor。

假设我们在前面章节配置的机器ip为192.168.0.11，另外有四台机器192.168.0.12~15也需要加入区块链网络。Sawtooth支持两种配置方式：静态组网和动态组网。静态组网适合节点数较少，或者需要某种特定的网络拓扑结构的情况。动态组网则会由每个节点自动发现并连接其他节点。以下我们配置所有节点以动态组网的形式加入网络。

首先我们需要在其他四台机器上安装和配置好sawtooth软件包，以下命令是从上面的章节复制下来的：

$ sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys 8AA7AF1F1091A5FD
$ sudo add-apt-repository 'deb http://repo.sawtooth.me/ubuntu/1.0/stable xenial universe'
$ sudo apt-get update
$ sudo apt-get install -y sawtooth python3-sawtooth-poet-sgx
$ sudo sawadm keygen
$ sudo systemctl enable sawtooth-intkey-tp-python.service sawtooth-poet-validator-registry-tp.service sawtooth-rest-api.service sawtooth-settings-tp.service sawtooth-validator.service

如果为Poet on SGX还需按2.1说明另外安装SGX驱动，并按4.3.2创建SGX配置文件。

然后，我们需要修改validator的配置文件，让它们能够互相发现。Validator的配置文件的位置是/etc/sawtooth/validator.toml。第一台机器上的配置文件内容为：

bind = [
  "network:tcp://192.168.0.11:8800",
  "component:tcp://127.0.0.1:4004"
]
 
peering = "dynamic"
 
endpoint = "tcp://192.168.0.11:8800"
 
seeds = ["tcp://192.168.0.12:8800", "tcp://192.168.0.13:8800", "tcp://192.168.0.14:8800", "tcp://192.168.0.15:8800"]
 
scheduler = 'parallel'
 
network_public_key = 'wFMwoOt>yFqI/ek.G[tfMMILHWw#vXB[Sv}>l>i)'
network_private_key = 'r&oJ5aQDj4+V]p2:Lz70Eu0x#m%IwzBdP(}&hWM*'
 
minimum_peer_connectivity = 2

其中，seeds这行里包括了其他几台机器的IP，因此当前机器启动时，会自动尝试向列表中的这些IP询问其他节点的存在情况。当网络中的节点非常多，或者采用动态IP地址时，可以只在列表中包括少量长期在线的IP地址或域名，而无需列举所有IP。

对于其他几台机器，配置文件里相应的改动为：

...
  "network:tcp://本机对外的IP:8800",
  ...
endpoint = "tcp://本机对外的IP:8800"
...
seeds = [其他机器的IP列表]
...

现在我们依次在所有机器上创建对应的配置文件：

$ sudo vi /etc/sawtooth/validator.toml
$ sudo chmod 640 /etc/sawtooth/validator.toml
$ sudo chown root:sawtooth /etc/sawtooth/validator.toml

重启所有的机器，等待启动过程完毕后，用以下命令在每台机器上查看当前机器的对端节点列表：

$ curl http://127.0.0.1:8008/peers
